# Video Recording Checklist

## Overview

This checklist guides you through the complete video production process for the Depot CI demo videos. Use it to ensure nothing is missed.

---

## Phase 1: Pre-Production (1-2 weeks before)

### Script Finalization

- [ ] **Script approved** by stakeholders
- [ ] **Timing verified** (read aloud, check timing)
- [ ] **Visual cues identified** and marked `[VISUAL]`
- [ ] **Screen recordings planned** for each scene
- [ ] **Alternative endings** selected (if needed)

### Assets Preparation

#### Screen Recordings
- [ ] Depot CI interface (build process)
- [ ] Railway dashboard (build in progress)
- [ ] GitHub Actions workflow (baseline)
- [ ] Code editor (OpenClaw demo app)
- [ ] Terminal/shell (commands and output)
- [ ] Web browser (depot.dev, Railway website)

#### Graphics & Animations
- [ ] Logo versions (Depot CI, Railway)
- [ ] Comparison table templates
- [ ] Cost calculator animation
- [ ] Timeline/progress bar graphics
- [ ] Feature icons and badges

#### On-Screen Text
- [ ] Title cards
- [ ] Metric displays (numbers, percentages)
- [ ] URLs and call-to-action text
- [ ] Caption/subtitle files (if using)

### Equipment Setup

#### Hardware
- [ ] **Microphone** (USB or XLR) — test recording
- [ ] **Headphones** — for monitoring audio
- [ ] **Webcam/camera** (if showing face)
- [ ] **Lighting** — ring light or softbox
- [ ] **Green screen** (if using chroma key)

#### Software
- [ ] **Screen recording software** (CleanShot X, OBS, Loom)
- [ ] **Video editing software** (Final Cut, Premiere, DaVinci)
- [ ] **Audio editing software** (Audacity, GarageBand)
- [ ] **Teleprompter** (if using one)

### Environment Setup

- [ ] **Quiet recording space** identified
- [ ] **Background** clean and professional
- [ ] **Lighting** tested (no shadows, good exposure)
- [ ] **Desk** organized (only essentials visible)
- [ ] **Notification sounds** disabled on all devices

### Technical Setup

#### Screen Recording Settings
- [ ] Resolution: 1920x1080 (1080p) minimum
- [ ] Frame rate: 30fps (or 60fps for smoother motion)
- [ ] Cursor: visible, appropriately sized
- [ ] File format: MP4 or MOV

#### Audio Settings
- [ ] Sample rate: 48kHz
- [ ] Bit depth: 16-bit or 24-bit
- [ ] Format: WAV or high-quality MP3
- [ ] Microphone level tested (not peaking)

## Phase 2: Recording Day

### Before Recording

- [ ] **Voice warm-up** (5 minutes of vocal exercises)
- [ ] **Hydration** (room temperature water)
- [ ] **Script printed** (for reference)
- [ ] **Recording software open** and tested
- [ ] **All apps/sites** open and logged in:
  - [ ] Depot dashboard
  - [ ] Railway dashboard
  - [ ] GitHub (with workflows ready)
  - [ ] Code editor with demo app

### Scene Recording Checklist

Use this for each scene:

#### Scene: ___ (fill in scene name)

- [ ] **Screen recording started** before action
- [ ] **Room is quiet** (no background noise)
- [ ] **Script in view** (teleprompter or printed)
- [ ] **Take deep breath**
- [ ] **Begin recording**
- [ ] **Perform action** (speak clearly, moderate pace)
- [ ] **Pause** after takes (for editing flexibility)
- [ ] **Stop recording**
- [ ] **Review footage** (check for issues)
- [ ] **Re-take if needed** (mark best take)

### Recording Tips by Scene Type

#### Voiceover Scenes (no screen)
- [ ] Speak at moderate pace (~140 words/min)
- [ ] Emphasize **bold** words
- [ ] Mark [PAUSE] points
- [ ] Record multiple takes
- [ ] Vary emphasis for editing options

#### Screen Recording Scenes
- [ ] Smooth mouse movements
- [ ] Don't rush actions
- [ ] Hover briefly before clicking
- [ ] Wait for pages to load
- [ ] Scroll smoothly and deliberately

#### Comparison Scenes
- [ ] Both recordings ready
- [ ] Timings synchronized
- [ ] Clear visual distinction
- [ ] Final times visible

## Phase 3: Post-Production

### Organization

- [ ] **Project folder created** with structure:
  ```
  project/
  ├── raw-footage/
  ├── screen-recordings/
  ├── audio-files/
  ├── graphics/
  ├── music/
  └── exports/
  ```
- [ ] **Files organized** by scene/section
- [ ] **Best takes identified** (mark clearly)
- [ ] **Backup created** of all raw files

### Editing Checklist

#### Rough Cut
- [ ] **Script imported** as reference
- [ ] **All footage imported** and organized
- [ ] **Timeline created** following script
- [ ] **Clips arranged** in correct order
- [ ] **Basic transitions** between scenes
- [ ] **Timing verified** (matches script duration)

#### Audio Editing
- [ ] **Voiceover synced** to visuals
- [ ] **Background music** added and mixed
- [ ] **Audio levels normalized** (-16 LUFS for dialogue)
- [ ] **Silence removed** from takes
- [ ] **Breath sounds reduced** (if distracting)

#### Visual Enhancements
- [ ] **Text overlays** added (timing, costs, metrics)
- [ ] **Highlighting** applied to key areas
- [ ] **Logos and branding** added
- [ ] **Color correction** applied consistently
- [ ] **Motion graphics** rendered and placed

#### Final Polish
- [ ] **Transitions smooth** (0.3-0.5s, not jarring)
- [ ] **Pacing consistent** throughout
- [ ] **Key information** on screen long enough to read
- [ ] **No visual glitches** or artifacts
- [ ] **Call-to-action** clear and visible

### Quality Control

#### Technical Review
- [ ] **No audio clipping** or distortion
- [ ] **No sync issues** between audio and video
- [ ] **Export settings correct** (1080p, 30fps)
- [ ] **File size reasonable** (under 500MB for 3-4 min)
- [ ] **Format correct** for platform (MP4 for YouTube)

#### Content Review
- [ ] **Script followed** accurately
- [ ] **All claims accurate** (times, costs, metrics)
- [ ] **Contact info correct** (URLs, handles)
- [ ] **No typos** in on-screen text
- [ ] **Branding consistent** (colors, logos)

## Phase 4: Export & Distribution

### Export Settings

#### For YouTube (1080p)
```
Format: MP4
Codec: H.264
Bitrate: 8-10 Mbps (VBR, 2-pass)
Audio: AAC, 192kbps, 48kHz
Resolution: 1920x1080
Frame Rate: 30fps
```

#### For Social Media (720p)
```
Format: MP4
Codec: H.264
Bitrate: 3-5 Mbps
Audio: AAC, 128kbps, 48kHz
Resolution: 1280x720
Frame Rate: 30fps
```

## Common Issues to Avoid

### Audio Issues
- ❌ Echo or reverb → Record in carpeted room, use soft surfaces
- ❌ Background noise → Close windows, turn off fans/AC
- ❌ Low volume → Use microphone close (4-6 inches)
- ❌ Popping sounds → Use pop filter or speak past microphone

### Visual Issues
- ❌ Small text → Use 24pt minimum, test on phone
- ❌ Cluttered screen → Clean desktop before recording
- ❌ Fast mouse → Slow down, be deliberate
- ❌ Glitchy transitions → Keep them simple (0.3-0.5s)

### Content Issues
- ❌ Too much jargon → Explain technical terms
- ❌ Too fast → Speak at conversational pace
- ❌ Wrong timing → Read script aloud and adjust
- ❌ Inaccurate claims → Verify all numbers and metrics
